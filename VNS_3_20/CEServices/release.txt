Notes :
    IPv6
    ====  
    Routing legs are not sending Router advertisements (Solicited and unsolicited).
    This impacts router discovery process of any hosts connected to the VLAN on which a routing leg is associated.

    Routing legs are not responding to the ICMPv6 pings to the link local IPv6 address.
    Responding to link local address is mandatory for the neighbor discovery process on link.

    IPv6 stateless auto configuration is not supported on the routing legs.
    Due to this limitation, IPv6 address has to manually configured on the routing legs.
    Path MTU determination also depends on the same.

    Routing legs are not responding for the ping  to all nodes multicast address when interface VLAN is not available.
    IPv6 supporting devices are required to respond to ping for all nodes multicast address to enable their discovery in network.

Notes: 
   MPLS
   ====
   The MPLS feature set is under development and should be considered work-in-progress
   for the present release.

   These high-level features are currently implemented in the MPLS_TP API:
   - LSR: L-LSP, E-LSP, Uniform/Pipe mode
   - LER: MPLS pseudo-wire with/without Control Word
   - MEF E-LINE using PW as transport (work-in-progress)
   - MPLS TC-to-COS and COS-to-TC mappings

   A set of test commands are provided when using ICLI. These will be removed when
   the final MPLS/MPLS-TP command set is specified.
   
   Major functions not included in this release:
   - Protection
   - OAM
   - MPLS policing
   - MPLS multicast
   - MPLS Statistics
   - E-TREE, E-LAN
