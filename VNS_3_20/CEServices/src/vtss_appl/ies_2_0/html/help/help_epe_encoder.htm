Ethernet PCM Encoder Configuration Help

Configure EPE port on this page.

To configure the Ethernet PCM encoder Ethnet traffic can be converted to the EPE port then output as a HDLC or CH7-15 encoded PCM stream.

If packet ONLY need to be encoded a mirror might need to be utilized to encode traffic. The switch may disgard packets being sent to the FPGA port. If the traffic is bi-directional through the EPE port , the mirror cannot be used.

The traffic to be copied on the EPE port is selected as follows:

  • Set the destination port of the mirror to the EPE port, port 14 on the 12 iES-12 and port 18 on the iES-16.
  • All frames received on a given port (also known as ingress or source mirroring).
  • All frames transmitted on a given port (also known as egress or destination mirroring).

Bit Rate

Bit Rate sets the bitrate to 200000-50000000 bps.

EPE Port

EPE Port Enables encoding in ISO/IEC 13239 HDLC Ethernet or CH7-15, IRIG 106-15 chapter 7 Ethernet formats.

CH7-15: Word(s) in Minor Frame

CH7-15: Word(s) in minor frame sets the number of data words in a minor frame. Values 1-8 may be used.

Invert Clock

The invert clock feature allows a device to flip the sample clocks polarity, optimizing PCM data decoding by aligning the sampling edge with the data signal.
